
初始化版本库时会发生什么情况？为什么你需要这样做？

	会新建一个.git文件夹，但是不会主动创建一个commit。初始化时不主动创建commit一是因为可能不需要commit整个文件夹，二是因为主动创建commit后无法为此次commit提供信息。


暂存区与工作目录和版本库有何不同？你认为它具有什么价值？

	暂存区是工作目录和版本库中间的一块缓存区域，通过它可以灵活的创建单个文件或多个文件的commit版本。


如何使用暂存区确保为每项逻辑更改进行一次提交？

	使用 git diff 和 git diff --staged 进行对比。


在哪些情况下，分支可以帮助你有序地组织历史记录？分支将如何提供帮助？	

	在不想对主程序产生影响的情况，对现有程序进行增强性开发时，分支能提供有序的组织历史记录，可以通过简单的checkout分支名在多个不同功能的程序之间进行切换。


图表如何帮助你直观显示分支结构？

	分层次显示。


将两个分支合并到一起有何结果？为什么我们在图表中表示它的方式是那样的？

	两个分支的commit按照提交时间顺序排列。


Git 的自动合并与始终手动进行合并各有何优缺点?

	不存在争议的合并使用自动合并很方便，遇到冲突手动解决后重新提交再合并，保持鲁棒性。
